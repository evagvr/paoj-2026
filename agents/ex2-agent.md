# Agent Rules: Laboratory 06 — Exercise 2 (Main, Multi-Part, Expands Example)

## Purpose
- Main exercise of the laboratory, building on the concepts from exercise1.
- Introduces new features and complexity, split into multiple parts (partA, partB, partC).

## Structure
- One Main.java file.
- Multiple parts, each with its own requirement and increasing complexity.
- Each part has its own input/output specification and example.
- Classes and interfaces may be reused and extended from exercise1.
- **Each part must have explicit test cases in a `tests/partX/` directory, with matching `.in` and `.out` files.**

## Test File Naming and Structure
- For each part, create a directory: `tests/partA/`, `tests/partB/`, `tests/partC/`.
- Each test case consists of a pair of files: `N.in` (input) and `N.out` (expected output), where N is a number (e.g., 1, 2, ...).
- Input and output must match the specification for each part exactly.
- Tests should cover normal, edge, and error cases where appropriate.

## Writing Style
- Each part is clearly separated in the Readme.
- Input/output examples for each part.
- Hints for new Java features or patterns introduced in each part.
- Encourages code reuse and extension.
- **Explicitly mention the existence and location of test cases in the Readme.**

## Grading
- Automatic, pass/fail per part, based on the provided test cases.
- Main points for the laboratory are awarded here.
- No bonus points, but full completion is required for maximum grade (except if bonus is used).

## Typical Requirements
- Implement a callback interface or strategy pattern.
- Use functional interfaces or lambdas if appropriate.
- Add new features or requirements in each part (e.g., new mathematical functions, new sorting criteria).
- Input/output must match the specification for each part.
- **Test cases are mandatory and must be provided for each part.**

## Example
- See laboratory06/exercise2/Readme.md for a canonical example.
- See `tests/partA/`, `tests/partB/`, `tests/partC/` for test case structure.
